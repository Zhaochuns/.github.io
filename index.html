作用域

  • 形参都是局部的

  • 不可变数据对象，在函数内外的值，用到了引用计数

  • 不同作用域之间的变量使用

  • 全局变量 的可以被 局部作用域使用

  • 但是局部变量不可以被全局作用域所使用，除非你用global语句升级，声明为全局变量

  • 想要函数内的代码生效,必须调用函数

函数内部如果直接使用全局可变对象

  • 全局可变对象,在函数内部使用的时,其实是一个共享状态
  • 是因为列表的指针域 是一个 也会存在引用计数的情况
  • 解决办法就是在函数内部使用深浅拷贝

可变数据在传递参数时

  • 在函数内部直接通过形参修改,也会影响原有数据
  • 可变对象在传递参数时,只是一个引用

不可变数据在传递参数时

  • 在函数内部直接通过形参修改,不会影响原有数据

匿名函数

  • lambda 表达式

  • 除了def语句,我们还可以通过lambda语句来创建函数

  • lambda创建的函数 因为默认不具有名字，他需要我们来指定一个变量名来保存

  • 表达式的结果，将作为返回值自动返回

  • 轻量级的函数 一般使用lambda来实现

函数

  • 按照过程编写代码，一般功能都是一次性的，非常不好维护
  • 把功能封装集成，方便二次开发和维护

  • def 语句 定义函数的语句

    • def 语句可以出现在整个Python脚本的任意地方

    • 函数内部重定义自己

  • 函数的参数

    • 函数的参数是在定义函数时括号中的内容

    • 形参 是为了方便在提前编写函数时，可以先在函数内部使用，挖个坑

      • 形参你可以想成是一个待赋值的变量

    • 实参 就是函数实际调用时传递进去的值

  • 参数探究

    • 必备参数

      • 当你定义函数时，形参已经有了，在实参传递的过程，一定要和形参一一对应

    • 命名参数 当函数参数太多了，记不住顺序呀 可以指定的写明形参名去赋值

    • 缺省参数

      • 可以在函数定义时，指明一个形参的默认值，就可以不给这个具有默认值的形参传递实参了

      • 当你给一个具有默认参数的形参传递一个实参，那么会覆盖掉原有的默认值

      • 缺省参数的定义顺序一定是从右向左，从后向前

    • 不定长参数

      • 元组不定长 *arg

        • *号 只是在定义形参时做标识的

      • 字典不定长 **kwargs

        • 这种不定长传参一定要 命名不定长

      • 联合使用

        • 不定长参数可以接收空值

文件操作下

每一次对文件的访问都是访问磁盘中的

fp = open() -> 返回了一个文件句柄

​ r：读文件

read(字符数) split(‘\n’)

readline()

readlines() 返回了一个文件中的字符串列表，以一行行的格式存储，那么这里列表就是每一行

fp.close() 关闭文件句柄 释放资源 Linux下面一个进程能打开的最大文件数是有上限的

​ 当你的程序结束的时候，进程就没了，那么进程所

  • 写文件：读写公用同一个指针

    • w：write 改权限可以让我们打开一个文件，进行写入，但是每一次打开都会将之前的文件内容清空
      • w可以清空文件
    • a：append 追加，读写指针在文件末尾

  • 缓冲区( 内存(临时) ) 针对读写文件

    • 保护设备
    • 让效率慢的和效率快的组件之间可以协同工作，缓和速度之间的矛盾

    磁盘的寿命由什么决定：写次数

    • 当你关闭文件的时候，缓冲区中的内容会刷新到磁盘上
    • 手动刷新缓冲区：fp.flush()

  • 缓存：为了提高数据流通速度

    缓存是在CPU里有

    缓存是一块超级告诉的数据流通空间（类似内存），但是太贵了，所以电脑上缓存空间一般不大

    L1：CPU经常从L1中拿取数据、控制线程和进程

    L2：存储数据

    L3：管控内存

  • 读写指针修改：fp.seek()

    • Python中文件读写指针共享

  • 读写指针位置获取：fp.tell()

  • 权限的总结

    • r+
    • w+
    • a+

    当你对权限具备加号，那么它将补充没有的权限

    • rb
    • wb
    • ab

    分别以二进制权限打开文件，一般处理：图像，音频，视频

  • 不同权限下读写指针位置：

    • r: 文件开头
    • w: 文件开头
    • a: 文件末尾

异常

  • 异常是我们手抖不细心，没有好好听课而导致写代码时的错误

  • 现在只要触发了异常，程序将终止

  • 当我们在做一些网络编程的时候，网络情况不一定什么时候是什么样子

  • 异常捕获 可以在错误发生时，捕捉到，默认的Python的行为将不执行

IndentationError: unexpected indent

​ 缩进错误

ZeroDivisionError: division by zero

​ 除数为0

NameError: name ‘b’ is not defined

​ 访问未声明变量

IndexError: list index out of range

​ 访问越界(超过了原有数据的长度)变量

KeyError: 2

​ 访问字典中(JSON类似)不存在key值

AttributeError: module ‘socket’ has no attribute ‘create’

​ 访问一个对象(变量，函数，类，模块)不存在的属性

  • 异常捕获可以提高代码的健壮性
  • 让我们的代码在不同情况下，可以让程序继续向下，而不是直接中断
  • 有一些异常错误，需要额外导入模块才可以使用，切记。
